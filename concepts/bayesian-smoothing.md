# Bayesian Smoothing

피드백 개수가 적을 때 극단적인 비율이 나오는 걸 막아주는 통계 기법이다. 예를 들어 어떤 문서가 딱 1번 인용됐는데 그 1번이 👍였다면, 단순 비율로는 "100% 만족도"가 된다. 이 문서를 피드백 10번 중 8번 👍인 다른 문서보다 위에 두는 건 직관적으로 이상하다 — 표본이 너무 적기 때문이다.

## 공식

```
score = (👍 개수 + 1) / (전체 반응 개수 + 2)
```

이건 Laplace smoothing(가상의 👍 1개, 👎 1개를 미리 깔아두는 것)과 같은 개념이다. 피드백이 0건이면 score는 0.5(중립)에서 시작하고, 피드백이 쌓일수록 실제 비율에 점점 가까워진다.

| 상황 | 단순 비율 | Bayesian smoothing |
|---|---|---|
| 👍 1, 👎 0 | 100% | 66.7% |
| 👍 8, 👎 2 | 80% | 75% |
| 👍 0, 👎 0 | 정의 불가 | 50% |

## 이 프로젝트에서

각 청크(chunk)의 `feedback` 가중치를 이 방식으로 계산해서, [Ranking Service](/notes/mattermost-second-brain-plan)의 `score = sim × time_decay × authority × feedback` 공식에 넣는다. 여기에 추가로:

- **시간 감쇠(time decay)** — 오래된 피드백일수록 비중을 줄여, 문서 내용이 바뀐 뒤에도 옛날 반응이 계속 영향을 주지 않게 한다
- **유저 다양성 보정** — 같은 사람이 반복해서 반응한 건 한 명의 의견으로 취급해 소수 의견 쏠림을 막는다
